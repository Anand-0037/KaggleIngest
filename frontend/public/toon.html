TOON Format v2.0

Token-Oriented Object Notation (TOON) is a production-ready data format that achieves 30-60% token reduction compared to JSON while improving LLM accuracy by 3-20% across different task types.

Why TOON?

  • 30-60% fewer tokens compared to JSON
  • Human-readable tabular structure
  • Optimized for LLM parsing (ChatGPT, Claude, Gemini)
  • Full round-trip JSON compatibility
  • Explicit structure helps LLMs validate data

Key Features

1. Tabular Arrays (TOON's Sweet Spot)

Declare field names once, stream rows as comma-separated values:

notebooks[2]{index,title,author,upvotes}:
  1,Titanic Data Science Solutions,Manav Sehgal,12345
  2,Ensemble Modeling,Yassine Ghouzam,10234

2. Nested Objects with Indentation

Like YAML, uses whitespace instead of braces:

metadata:
  title: Kaggle Competition
  url: https://www.kaggle.com/c/titanic
  status: Active

3. Smart Quoting

Quotes only when necessary:

  • hello world - Unquoted (inner spaces OK)
  • "hello, world" - Quoted (contains delimiter)
  • " padded " - Quoted (leading/trailing spaces)

When to Use TOON

Best For

  • Large tabular datasets (50-60% savings)
  • LLM prompts with structured data
  • Time-series data (59% reduction)
  • Budget-conscious LLM applications

Consider Alternatives For

  • Pure CSV data (CSV is ~5% smaller)
  • Deeply nested structures
  • Existing JSON pipelines

Quick Start

JavaScript/TypeScript

import { encode, decode } from '@toon-format/toon'

const toon = encode({ users: [...] })
const data = decode(toon)

Python

from toon import encode, decode

toon = encode({"users": [...]})
data = decode(toon)

CLI (No Installation)

npx @toon-format/cli input.json -o output.toon --stats
