TOON Format v2.0

Token-Oriented Object Notation (TOON) is a compact, LLM-friendly data format designed to save context window space.

Why TOON?

  • üöÄ 30-60% fewer tokens compared to JSON
  • üìñ Human-readable tabular structure
  • ü§ñ Optimized for LLM parsing (ChatGPT, Claude, Gemini)

Example

metadata{Title,URL,Description,Status}
Titanic - Machine Learning from Disaster,https://www.kaggle.com/c/titanic,Predict survival,Active

schema{File,Columns}
train.csv,[PassengerId (int64), Survived (int64), Pclass (int64)]

notebooks{index,title,author,upvotes}
1,Titanic Data Science Solutions,Manav Sehgal,12345
2,Ensemble Modeling,Yassine Ghouzam,10234

statistics{total_requested,successful,failed}
5,5,0

Specification

TOON uses a header-row structure similar to CSV but grouped by sections. Each section starts with section_name{columns} followed by data rows.
